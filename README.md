# Автоматизированный ИИ-оркестратор (Jira - Obsidian)

Автономный фоновый сервис (оркестратор), который собирает данные о работе кросс-функциональной команды из таск-трекера Jira, анализирует их с помощью LLM (через паттерн Agentic Loop) и детерминированно модифицирует локальную графовую базу знаний Obsidian.

Система разработана для руководителей ИТ-команд, позволяя автоматизировать сбор контекста и управление знаниями, сохраняя существующую разметку и метаданные в файлах.

## 🌟 Основные возможности (MVP)

Приложение реализует три продуктовых среза через консольный интерфейс:

* **Авто-Standup (Daily-ассистент):** Анализирует дельту за последние 24 часа. LLM выделяет реальный прогресс, текущий фокус (задачи в работе) и аномалии (баги, блокировки, задержки). Сводка записывается в секцию H2 ежедневной заметки (формат `YYYY-MM-DD.md`).


* **Стратегический статус (Эпики):** Оркестратор самостоятельно считывает локальный файл приоритетов `priorities.md` и фильтрует поток задач Jira. Подсвечивается прогресс исключительно по целевым эпикам.


* **Ретроспектива и итоги спринта:** Агрегация данных завершающегося цикла. Алгоритм сравнивает фактический результат с заявленными целями из файла `goals.md`. Выявляются добавленные после старта задачи (Scope Creep) и формируется список кандидатов на перенос.



## 🏗 Архитектура и технологии

Ядро системы — Python-оркестратор, который выступает MCP-клиентом (Model Context Protocol) и общается с LLM в цикле ReAct.

* **LLM и Агент:** Модель `qwen/qwen3-next-80b-a3b-thinking`, доступная через OpenRouter API. Цикл рассуждений реализуется классом `AIAgent`, очищающим финальные ответы от внутренних тегов `<think>`.


* **Jira MCP Server (`jira_mcp.py`):** Извлекает контекст из трекера по JQL-запросам. Инструменты умеют собирать сводки по статусам, исполнителям, меткам и последним комментариям к задачам.


* **Obsidian MCP Server (`obsidian_mcp.py`):** Выполняет функцию AST-инжектора. Используя библиотеку `markdown-it-py`, сервер лексически анализирует Markdown, находит нужные заголовки H2 и безопасно заменяет контент. Это обеспечивает идемпотентность выполнения и сохранность YAML-метаданных.



## 📂 Структура проекта

* `main.py` — Точка входа, запускающая асинхронный цикл и инициализирующая агента.


* `app.py` — Главный класс консольного приложения с выбором режима генерации отчетов.


* `agent.py` — Оркестратор цикла ReAct и менеджер MCP-серверов.


* `prompts.py` — Менеджер системных промптов для жесткого контроля форматов LLM.


* `config.py` — Конфигурация API и выбор LLM.


* `jira_mcp.py` и `obsidian_mcp.py` — Локальные MCP-серверы для интеграции с внешними инструментами.


* `mock_obsidian_vault/` — Изолированная директория для тестирования записи в Markdown-файлы (содержит `goals.md`, `priorities.md` и ежедневные заметки).


* Папка `scripts/` — Набор утилит (`jira_seeder.py`, `jira_deleter.py`) для генерации и очистки фейкового датасета тикетов на основе JSON-мока `mock_sprint.json`.



## 🚀 Инструкция по установке и запуску

### 1. Подготовка окружения

Убедитесь, что у вас установлен Python (проект тестировался на Python 3.12).
Склонируйте репозиторий и установите зависимости:

```bash
git clone https://github.com/verarod1/jira-obsidian-orchestrator.git
cd jira-obsidian-orchestrator
python -m venv venv
# Для Windows: .\venv\Scripts\activate
# Для Linux/macOS: source venv/bin/activate
pip install -r requirements.txt

```

Список ключевых зависимостей включает `mcp`, `requests`, `python-dotenv` и `markdown-it-py`.

### 2. Настройка переменных среды

Создайте файл `.env` в корневой директории проекта и добавьте следующие конфигурационные ключи:

```env
# Доступ к Atlassian Jira Cloud
JIRA_URL="https://ваш-домен.atlassian.net"
PROJECT_KEY="ключ_вашего_проекта"
JIRA_EMAIL="ваш_email@example.com"
JIRA_API_TOKEN="ваш_токен_atlassian"

# Доступ к LLM
OPENROUTER_API_KEY="ваш_токен_openrouter"
GROQ_API_KEY="ваш_токен_groq"

```

*(Токены загружаются в приложение через библиотеку `python-dotenv`)*.

### 3. Генерация тестовых данных (Data Seeding)

Для проверки пайплайна необходимо имитировать работу команды. Воспользуйтесь скриптами для сидирования:

* Запустите скрипт генерации (создаст задачи, эпики и раскидает статусы по доске из `mock_sprint.json`): `python scripts/jira_seeder.py`.


* Для очистки тестовой среды от сгенерированных тикетов используйте `python scripts/jira_deleter.py`.



### 4. Запуск приложения

Запустите основной интерфейс программы:

```bash
python main.py

```

Система предложит выбрать один из трёх режимов генерации аналитики, после чего ИИ-агент самостоятельно свяжется с серверами, соберет данные и запишет итоговый Markdown-отчет в локальную папку `mock_obsidian_vault/`.
